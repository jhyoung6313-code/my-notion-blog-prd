## 1. 개요 (Overview)

| 항목 | 내용 |
|------|------|
| 프로젝트명 | **개인 개발 블로그 (Dev Blog with Notion CMS)** |
| 한 줄 요약 | Notion에서 글을 작성하면 자동으로 웹사이트에 발행되는 개인 기술 블로그 |
| 목적 | Notion을 CMS로 활용해, 별도 관리자 페이지나 DB 없이 글을 작성·관리하는 블로그를 운영한다 |
| 타깃 사용자 | ① **운영자**(나) — 글을 쓰는 사람, ② **방문자** — 글을 읽는 독자 |

### 1.1 CMS로 Notion을 선택한 이유

- **친숙한 글쓰기 경험**: 마크다운·이미지·코드 블록을 Notion 에디터에서 그대로 작성한다.
- **별도 백오피스 불필요**: 글 작성/수정/삭제 UI를 직접 만들 필요가 없다. Notion이 곧 관리자 화면이다.
- **무료 데이터베이스**: 글 메타데이터(제목·카테고리·태그·발행일)를 Notion 데이터베이스로 관리하므로 별도 DB 구축·운영 비용이 없다.
- **실시간 반영**: Notion에서 글을 수정하면 ISR(Incremental Static Regeneration)로 웹사이트에 자동 반영된다.
- **협업 확장 용이**: 추후 팀 블로그로 확장 시 Notion 공유만으로 기여자를 늘릴 수 있다.

## 2. 목표와 비목표 (Goals & Non-Goals)

### 2.1 목표 (Goals)

- G1. Notion 데이터베이스의 글 목록을 가져와 홈에 카드 형태로 보여준다.
- G2. 개별 글의 Notion 페이지 본문을 렌더링하는 상세 페이지를 제공한다.
- G3. 카테고리·태그로 글을 필터링한다.
- G4. 제목·요약 기반 검색을 제공한다.
- G5. 모바일·데스크톱 반응형 디자인을 지원한다.

### 2.2 비목표 (Non-Goals — MVP에서 제외)

- 댓글 시스템 (추후 Giscus 등으로 확장)
- 좋아요·조회수 등 인터랙션 집계
- 다국어(i18n)
- 로그인·회원 기능 (블로그는 공개 읽기 전용)
- RSS·뉴스레터 구독

## 3. 주요 기능 (Features)

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|----------|
| F-01 | 글 목록 조회 | Notion DB에서 `Status = 발행됨` 인 글을 발행일 내림차순으로 가져온다 | P0 (MVP) |
| F-02 | 글 상세 페이지 | 개별 글의 Notion 블록 본문(텍스트·이미지·코드)을 렌더링한다 | P0 (MVP) |
| F-03 | 카테고리 필터 | 카테고리(select)별로 글 목록을 좁힌다 | P0 (MVP) |
| F-04 | 태그 표시·필터 | 태그(multi_select)를 글에 표시하고 태그별 필터링한다 | P1 |
| F-05 | 검색 | 제목·요약 텍스트 부분 일치 검색 (클라이언트 사이드) | P1 |
| F-06 | 반응형 디자인 | 모바일/태블릿/데스크톱 레이아웃 대응 | P0 (MVP) |
| F-07 | 다크 모드 | 라이트/다크 테마 토글 (localStorage 영속화) | P2 |
| F-08 | SEO 메타 태그 | 글별 OG 태그·메타 디스크립션 생성 | P1 |

## 4. 기술 스택 (Tech Stack)

| 영역 | 선택 | 비고 |
|------|------|------|
| Frontend | **Next.js 15** (App Router) + **TypeScript** | 서버 컴포넌트 + ISR로 Notion 데이터 캐싱 |
| CMS | **Notion API** (`@notionhq/client`) | 글 메타데이터 + 본문 블록 조회 |
| 본문 렌더링 | `notion-to-md` 또는 `react-notion-x` | Notion 블록 → 마크다운/JSX 변환 |
| Styling | **Tailwind CSS** + **shadcn/ui** | 유틸리티 CSS + 접근성 좋은 컴포넌트 |
| Icons | **Lucide React** | 일관된 아이콘 세트 |
| Deployment | **Vercel** | Next.js 네이티브 호스팅, ISR/환경 변수 지원 |

### 4.1 환경 변수

```bash
# .env.local
NOTION_API_KEY=secret_xxxxxxxxxxxxxxxx     # Notion Integration Token
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxx        # 블로그 글 데이터베이스 ID
```

> ⚠️ 위 키는 절대 커밋하지 않는다. `.env.local`은 `.gitignore`에 포함한다.

## 5. Notion 데이터베이스 구조 (Schema)

블로그 글 하나 = Notion 데이터베이스의 한 행(페이지).

| 필드명 | Notion 타입 | 설명 | 필수 |
|--------|-------------|------|------|
| `Title` | title | 글 제목 | ✅ |
| `Slug` | rich_text | URL 경로 (예: `my-first-post`). 비면 제목에서 자동 생성 | ✅ |
| `Category` | select | 카테고리 (예: Frontend / Backend / 회고) | ✅ |
| `Tags` | multi_select | 태그 (예: React, TypeScript) | ⬜ |
| `Summary` | rich_text | 목록 카드에 노출할 한 줄 요약 | ⬜ |
| `Cover` | files / url | 썸네일 이미지 | ⬜ |
| `Published` | date | 발행일 (정렬 기준) | ✅ |
| `Status` | select | `초안` / `발행됨` — `발행됨`만 사이트 노출 | ✅ |
| `Content` | (페이지 본문) | 글 본문 블록 (Notion 에디터로 작성) | ✅ |

## 7. MVP 범위 (Scope)

MVP는 **"Notion에 쓴 글이 블로그에 보이고, 클릭하면 읽힌다"** 까지를 목표로 한다.

**포함 (P0):**
- Notion API 연동 및 환경 설정
- 글 목록 페이지 (F-01)
- 글 상세 페이지 (F-02)
- 카테고리 필터 (F-03)
- 기본 스타일링 + 반응형 (F-06)

**다음 단계 (P1 이후):**
- 태그 필터 (F-04), 검색 (F-05), SEO (F-08), 다크 모드 (F-07)

## 9. 데이터 흐름 (Data Flow)

```
[운영자] Notion에서 글 작성 (Status=발행됨)
            │
            ▼
[Next.js 서버] 빌드/ISR 시점에 Notion API 호출
   - databases.query  → 글 목록 (메타데이터)
   - blocks.children.list → 글 본문 블록
            │
            ▼
[정적 페이지 생성 + 캐시(ISR revalidate)]
            │
            ▼
[방문자] 브라우저에서 블로그 열람
```

- **캐싱 전략**: ISR `revalidate: 60`(초) — 글 수정 후 최대 1분 내 반영. Notion API 호출 비용·레이트리밋 절감.

## 10. 성공 지표 (Success Metrics)

- Notion에서 글을 발행하면 **1분 이내** 사이트에 반영된다.
- 홈/상세 페이지 **Lighthouse 성능 90점 이상**.
- 글 추가 시 코드 수정 없이 **Notion 작성만으로** 발행 가능하다.

## 11. 리스크 및 대응 (Risks)

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Notion API 레이트리밋 (초당 3req) | 빌드 지연 | ISR 캐싱 + 요청 배치/지연 |
| Notion 블록 타입 미지원 | 본문 깨짐 | `react-notion-x`로 폭넓은 블록 지원, 미지원 블록 폴백 처리 |
| API 키 노출 | 보안 사고 | 서버 컴포넌트에서만 호출, `.env.local` gitignore |
| Notion 서비스 장애 | 사이트 데이터 누락 | ISR 캐시로 마지막 정상 데이터 유지 |
